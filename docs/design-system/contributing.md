# Contributing to Design System\n\nDesign System에 새로운 컴포넌트를 추가하거나 기존 컴포넌트를 수정할 때의 절차와 체크리스트입니다.\n\n## 컴포넌트 추가 절차\n\n### 1단계: 컴포넌트 설계\n\n새 컴포넌트를 만들기 전에:\n\n- 기존 컴포넌트로 조합 가능한지 검토\n- Props 인터페이스 설계 (명확한 이름, 필수/선택 구분)\n- 사용 사례 3개 이상 정의\n- 접근성 고려 (ARIA, keyboard nav, focus)\n\n```tsx\n// 예: 새로운 Selector 컴포넌트\nexport type SelectorSize = 'sm' | 'md' | 'lg'\n\nexport interface SelectorProps {\n  items: Array<{ id: string; label: ReactNode }>\n  value: string\n  onChange: (id: string) => void\n  placeholder?: string\n  disabled?: boolean\n  size?: SelectorSize\n}\n```\n\n### 2단계: 파일 생성\n\n경로: `src/renderer/src/components/common/ds/YourComponent.tsx`\n\n```tsx\nimport { forwardRef } from 'react'\nimport type { ReactNode } from 'react'\n\nexport type YourComponentVariant = 'primary' | 'secondary'\n\nexport interface YourComponentProps {\n  variant?: YourComponentVariant\n  children: ReactNode\n  className?: string\n}\n\n/** 한 줄 설명\n *  사용: <YourComponent variant=\"primary\">콘텐츠</YourComponent>\n */\nfunction YourComponent({\n  variant = 'primary',\n  className = '',\n  children,\n  ...rest\n}: YourComponentProps): JSX.Element {\n  const v = variant === 'primary' ? '' : ` ${variant}`\n  return (\n    <div className={`ds-your-component${v} ${className}`} {...rest}>\n      {children}\n    </div>\n  )\n}\n\nexport default YourComponent\n```\n\n### 3단계: CSS 클래스 추가\n\n`src/renderer/src/design-system.css`에 utility 클래스 추가:\n\n```css\n/* ---------- Your Component ---------- */\n.ds-your-component {\n  display: block;\n  padding: var(--space-2);  /* 토큰 사용 */\n  border-radius: var(--radius-md);\n  background: var(--bg-surface);\n  border: 1px solid var(--bg-border);\n}\n.ds-your-component.primary {\n  background: var(--c-blue-bg);\n  color: var(--c-blue-fg);\n}\n.ds-your-component.secondary {\n  background: transparent;\n  border-color: var(--c-orange-bg);\n  color: var(--c-orange-fg);\n}\n```\n\n### 4단계: 토큰 확인\n\n필요한 새 토큰이 있으면 `src/renderer/src/index.css`에 추가:\n\n```css\n/* 라이트 모드 */\n[data-theme='light'] {\n  --my-new-token: #FFFFFF;\n}\n\n/* 다크 모드 */\n[data-theme='dark'] {\n  --my-new-token: #1C2027;\n}\n```\n\n**체크리스트**:\n- [ ] 라이트 모드 값\n- [ ] 다크 모드 값\n- [ ] 명도 대비 >= AA (4.5:1)\n- [ ] 모드 전환 시 시각 테스트\n\n### 5단계: 타입 정의\n\n필요하면 `src/shared/types/`에 공용 타입 추가:\n\n```tsx\n// src/shared/types/components.ts\nexport type YourComponentSize = 'sm' | 'md' | 'lg'\nexport type YourComponentState = 'idle' | 'loading' | 'error'\n```\n\n### 6단계: 문서 작성\n\n`docs/design-system/components/YourComponent.md` 생성:\n\n```markdown\n# YourComponent\n\n한 줄 설명\n\n## 언제 쓰나\n- 사용 사례 1\n- 사용 사례 2\n\n## Props\n| 이름 | 타입 | 기본값 | 설명 |\n|------|------|--------|------|\n\n## 기본 사용\n```tsx\n// 예제\n```\n```\n\n## 명명 규칙\n\n### Props\n\n```tsx\n// ✓ 좋음\ninterface ButtonProps {\n  variant: 'primary' | 'secondary'  // 명확한 의도\n  size: 'sm' | 'md' | 'lg'  // 크기\n  onClick: () => void  // 콜백\n  disabled: boolean  // boolean\n  loading: boolean\n  icon: ReactNode  // 아이콘\n  children: ReactNode  // 콘텐츠\n}\n\n// ✗ 나쁨\ninterface ButtonProps {\n  type: string  // \"type\"은 HTML 속성명, 충돌 가능\n  size: number  // enum string이 나음\n  cb: () => void  // 약자 사용 금지\n  isDisabled: boolean  // \"disabled\"가 표준\n  content: ReactNode  // \"children\"이 React 표준\n}\n```\n\n### CSS 클래스\n\n```css\n/* ✓ 좋음 */\n.ds-button { }  /* 네임스페이스 */\n.ds-button.primary { }  /* 명확한 variant */\n.ds-button.sm { }  /* 단순 size */\n\n/* ✗ 나쁨 */\n.button { }  /* 충돌 위험 */\n.btn-primary { }  /* 일관성 없음 */\n.button-size-small { }  /* 너무 길게 */\n```\n\n### 토큰\n\n```css\n/* ✓ 좋음 */\n--c-blue-bg  /* 색상-톤-용도 */\n--c-blue-fg\n--c-blue-solid\n--elev-1  /* 카테고리-레벨 */\n--space-2\n\n/* ✗ 나쁨 */\n--button-bg  /* 컴포넌트 종속 */\n--blue-light  /* 모드별 구분 없음 */\n--shadow  /* 목적 불명확 */\n```\n\n## 라이트/다크 모드 검증\n\n### 색상 쌍 추가 시\n\n```tsx\n// 1. 라이트 모드\n[data-theme='light'] {\n  --my-bg: #E1EBFC;  /* 밝음 */\n  --my-fg: #1D4ED8;  /* 어두움 */\n}\n\n// 2. 다크 모드\n[data-theme='dark'] {\n  --my-bg: rgba(59,130,246,0.16);  /* 낮은 alpha */\n  --my-fg: #93C5FD;  /* 밝음 */\n}\n\n// 3. 명도 대비 검증 (https://webaim.org/resources/contrastchecker/)\n// 라이트: #E1EBFC vs #1D4ED8 → ~8:1 ✓\n// 다크: rgba(59,...,0.16) vs #93C5FD → ~5.5:1 ✓\n\n// 4. 시각 테스트\n// 라이트/다크 모드 모두에서 확인\n```\n\n## Props 베스트 프랙티스\n\n### 필수 vs 선택\n\n```tsx\ninterface BadProps {\n  // ✗ 나쁨: 모든 prop 필수\n  title: string\n  icon: ReactNode\n  onClick: () => void\n}\n\ninterface GoodProps {\n  // ✓ 좋음: 명확한 기본값\n  variant?: 'primary' | 'secondary'  // 기본값 있으면 선택\n  children: ReactNode  // 콘텐츠는 필수\n  onClick?: () => void  // 액션은 선택\n  disabled?: boolean  // boolean은 false 기본\n  size?: 'sm' | 'md' | 'lg'  // 기본값 있으면 선택\n}\n```\n\n### 기본값\n\n```tsx\n// ✓ 컴포넌트에서\nfunction Button({\n  variant = 'secondary',  // 기본값 명확\n  size,  // size는 optional\n  onClick,  // 콜백\n  disabled,\n  children\n}: ButtonProps) {\n  const sz = size && size !== 'md' ? ` ${size}` : ''  // md 생략\n  return (\n    <button\n      className={`ds-btn ${variant}${sz}`}\n      onClick={onClick}\n      disabled={disabled}\n    >\n      {children}\n    </button>\n  )\n}\n```\n\n## 접근성 체크리스트\n\n새 컴포넌트를 만들 때:\n\n- [ ] 키보드 네비게이션 (Tab, Enter, Esc)\n- [ ] 포커스 상태 (ring-focus 토큰)\n- [ ] ARIA 라벨 (aria-label, aria-describedby)\n- [ ] 색상 대비 (WCAG AA 이상)\n- [ ] 스크린 리더 지원\n- [ ] 마우스/터치/키보드 모두 지원\n\n### 예: 접근성 있는 Button\n\n```tsx\nfunction AccessibleButton({\n  variant,\n  onClick,\n  disabled,\n  ariaLabel,\n  children\n}: ButtonProps) {\n disabled 상태 명시\n    >\n      {children}\n    </button>\n  )\n}\n```\n\n## 테스트\n\n### 단위 테스트\n\n`src/renderer/src/components/common/ds/YourComponent.test.tsx`:\n\n```tsx\nimport { render, screen } from
