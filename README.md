# Clauday

> **두레이(Dooray) × Claude Code**, 한 창에 묶은 사내 오픈소스 AI 업무 비서

매일 두레이 태스크 확인하고, 메신저 훑고, 보고서 쓰고, 터미널에서 Claude Code 돌리는 게 반복되길래
"그냥 한 앱에서 다 되면 안 되나?" 싶어서 만든 Electron 앱입니다.
혼자 쓰기 아까워서 풀어요. 같이 쓰고 같이 개선해요. 🙂

![Clauday 대시보드](docs/screenshots/iShot_2026-04-23_19.18.33.png)

---

## 세 줄 요약

- 두레이(태스크·위키·캘린더·메신저·보고서)와 Claude Code를 한 Electron 앱에서 씁니다.
- 반복 업무는 AI가 요약·작성·스캔·정리해줍니다. 기능별로 **Haiku / Sonnet / Opus** 자동 라우팅.
- 스킬·MCP·와처·세션을 GUI로 관리하고 팀에 공유할 수 있어요.

## 1. 대시보드 — 오늘 할 일을 한 눈에

앱을 열면 가장 먼저 보이는 화면입니다. 받은 태스크가 수백 개라도 오늘 봐야 할 것만 추려서 맨 위에 띄웁니다.
"빠른 태스크 생성" 박스에 제목만 치면 담당자·마일스톤·태그는 AI가 채워요.

![대시보드](docs/screenshots/iShot_2026-04-23_19.18.33.png)

- **빠른 태스크 생성** — 제목만 입력 → 나머지 필드는 AI가 추론
- **오늘 집중 태스크** — 마감 임박 / 내가 담당 / 진행 중 기준으로 자동 정렬
- **통계 배너** — 전체 / 오늘 / 진행 / 지연 / 완료를 한 줄에

## 2. 태스크 · 프로젝트 사이드바

좌측 프로젝트 사이드바에서 프로젝트를 고르면 해당 프로젝트 태스크만 보입니다.
두레이 웹을 여러 탭 띄워두던 습관이 그냥 없어져요.

![태스크 + 프로젝트 사이드바](docs/screenshots/iShot_2026-04-23_19.20.54.png)

- **프로젝트별 필터** — 사이드바 클릭 한 번으로 해당 프로젝트 태스크만
- **상태 · 담당자 · 마일스톤** 기본 필터 — Dooray API와 1:1 호환
- **인라인 편집** — 목록에서 바로 상태/담당자 수정

## 3. 위키 — AI 구조 분석

프로젝트 위키를 트리로 펼치면서, AI가 "이 위키 중복이에요", "이 카테고리 문서가 비어 있어요" 같은 걸 알려줍니다.
신규 입사자용 온보딩 맵도 버튼 하나로 뽑을 수 있어요.

![위키 AI 분석](docs/screenshots/iShot_2026-04-23_19.22.16.png)

- **커스텀 순서** — 위키 트리 드래그로 직접 정렬
- **AI 구조 분석** — 중복 / 결락 / 고아 문서 리포트
- **AI 요약** — 긴 문서 상단에 TL;DR 자동 삽입

## 4. 캘린더 — AI 일정 분석

단순 뷰어가 아니라 "오늘 3시간 공백이에요", "화요일 회의 7개 — 번아웃 주의", "15~18시 연속 회의" 같은 분석을 자동으로 넣어줍니다.
빈 시간 자동 추천은 생각보다 많이 쓰게 됩니다.

![캘린더 AI 분석](docs/screenshots/iShot_2026-04-23_19.23.31.png)

- **빈 시간 찾기** — 당일 공백 블록을 AI가 추천
- **바쁜 날 경고** — 회의 4개 이상이면 경고 배지
- **연속 회의 탐지** — 이동/휴식 없는 일정을 분리 제안

## 5. 메신저 — AI 대화 요약 & 작성

수십 개 채널 다 읽는 건 현실적으로 불가능하잖아요. 놓친 멘션/질문을 추려주고,
"회식 장소 추천" 같은 주제가 뜨면 발언자·후보·결론을 표로 정리합니다.

작성 쪽은 **에이전틱 웹 조사 모드**로 동작해요. "다음 주 점심 장소 추천해줘" 같은 요청에
AI가 `WebSearch` / `WebFetch`로 직접 외부 사실을 조사한 뒤 메시지에 반영합니다.

| 채널 목록 · AI 설정 | 대화 주제 자동 정리 | AI 설정 모달 |
|---|---|---|
| ![메신저 채널](docs/screenshots/iShot_2026-04-23_19.29.33.png) | ![AI 정리](docs/screenshots/iShot_2026-04-24_14.23.17.png) | ![AI 설정](docs/screenshots/iShot_2026-04-24_14.23.30.png) |

## 설치 & 개발

### 요구사항

- Node.js 20+
- Python 3.11 (node-gyp 호환)
- macOS / Windows
- Claude Code CLI가 설치되어 있고 로그인된 상태

### 개발 서버

```bash
npm install
npm run dev
```

### 빌드

```bash
npm run dist       # macOS dmg
npm run dist:win   # Windows exe
npm run dist:all   # macOS + Windows
```

빌드 결과물은 `release/` 디렉터리에 생성됩니다.

## 릴리즈

릴리즈는 **태그 푸시 트리거**입니다. main에 머지만 해서는 릴리즈가 나가지 않아요.

```bash
# 버전 올리고 태그 푸시
git tag v1.2.1
git push origin v1.2.1
```

그러면 `.github/workflows/release.yml`이 돌면서:

- macOS (dmg, zip) 빌드 → GitHub Release에 자동 업로드
- Windows (exe) 빌드 → GitHub Release에 자동 업로드
- 릴리즈 노트 자동 생성

수동 실행도 가능합니다: GitHub → Actions → Release → **Run workflow**.

> Apple Developer 인증서 시크릿(`MAC_CSC_LINK`, `APPLE_ID` 등)이 등록되어 있으면 자동 서명, 없으면 unsigned dmg가 나옵니다.

## 로드맵

대략 이렇게 키워 나갈 예정입니다. 우선순위는 피드백 받으면서 조정할게요.

- **Phase 1 · AI 업무 대시보드** `작업 중` — 태스크 요약, 자연어 관리, 스마트 알림
- **Phase 2 · AI 업무 보고** — 일일/주간 보고서, 위키 자동화, 회의록
- **Phase 3 · Claude Code 통합** — 태스크 ↔ 코드 브릿지, AI 코드리뷰 → 코멘트
- **Phase 4 · 팀 인사이트** — 워크로드 시각화, 프로젝트 건강도, 스프린트 리포트

## 기여하기

사내 오픈소스입니다. 혼자 만든 거라 버그도 많고, 쓰다 보면 "이거 왜 이렇게 돼?" 하는 부분도 분명 있을 거예요.

**이렇게 도와주시면 좋아요**

- 써보고 "여긴 이래서 불편하다" 한 줄 피드백
- 자기 팀 업무 패턴을 스킬/와처로 만들어 커뮤니티에 공유
- "이 기능도 있으면 좋겠다" 제안 (이슈/메신저 모두 OK)
- 버그 리포트 — 재현 경로 + 스크린샷이면 충분합니다
- PR 환영. 커밋 단위 작게 쪼개서 주시면 리뷰 빠르게 할게요
